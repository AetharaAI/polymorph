# Bootstrap Templates

These files are starter policy documents for new projects.

Use them when bootstrapping a repo that needs:
- a repo-level rules file
- memory/state guidance
- provider-specific assistant instructions

Recommended usage:
1. copy `AGENTS.md` to the repo root
2. keep `MEMORY.md` near project state docs or repo root
3. copy provider-specific files only if that assistant/IDE uses them
4. update `project-state/ai/current-state.yaml` first after bootstrapping
