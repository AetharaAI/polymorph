# Deterministic Security Interceptor Pipeline

## 1. Ordered Pipeline Stages

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    DETERMINISTIC DECISION PIPELINE                      │
│                         Total Budget: 15ms P99                          │
└─────────────────────────────────────────────────────────────────────────┘

Stage 0: Pre-Processing (5ms)
    ├─ [0.1] Request Ingestion & Timestamping
    ├─ [0.2] Protocol Parsing (HTTP/2, gRPC, or custom binary)
    └─ [0.3] Request ID Generation (UUIDv7 for ordering)

Stage 1: Canonicalization (3ms)
    ├─ [1.1] Header Normalization (lowercase, whitespace trim)
    ├─ [1.2] URL Canonicalization (remove fragments, decode percent-encoding)
    ├─ [1.3] Body Serialization (JSON key sorting, protobuf canonical encoding)
    └─ [1.4] Hash Computation (SHA-256 over canonical form for replay)

Stage 2: Parallel Enrichment (4ms) ─── ALL MUST COMPLETE WITHIN DEADLINE
    ├─ [2.1] Identity Resolution (source/destination agent verification)
    ├─ [2.2] Policy Version Lookup (fetch versioned rule set)
    ├─ [2.3] Threat Intel Cache Check (IP reputation, agent risk score)
    └─ [2.4] Rate Limit State (per-agent counter from shared cache)

Stage 3: Rule Evaluation (2ms)
    ├─ [3.1] Rule Set Selection (based on policy version)
    ├─ [3.2] First-Match-Wins Evaluation (ordered rule list)
    └─ [3.3] Decision Determination (ALLOW/BLOCK/CONSTRAIN)

Stage 4: Action Application (1ms)
    ├─ [4.1] CONSTRAIN Modifications (header strip, body truncate, etc.)
    ├─ [4.2] Decision Logging (append-only audit log)
    └─ [4.3] Response Formatting (decision + metadata for downstream)

Stage 5: Output & Forwarding (<0.5ms)
    └─ [5.1] Forward to Destination or Return Decision Response
```

## 3. Failure-Mode Behavior Per Stage

### Stage 0: Pre-Processing Failures

| Failure Mode | Detection | Mitigation | Decision Impact |
|--------------|-----------|------------|-----------------|
| **Malformed protocol** | Parse exception within timeout | Log error, return `BLOCK` with `reason=parse_error` | BLOCK (fail-closed) |
| **Timeout exceeded** | Wall-clock >5ms budget enforced by watchdog thread | Kill request processing thread, return `BLOCK` with `reason=timeout` | BLOCK (fail-closed) |
| **Missing required headers** | Header validation fails after canonicalization | Return `BLOCK` with `reason=missing_header:<name>` | BLOCK (fail-closed) |
